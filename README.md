# Multimodal Emotion Recognition

A deep learning project that recognizes human emotion from **speech audio** and **text** using state-of-the-art transformer models — implemented as two independent, fully-trained pipelines.

## Objective

This project implements two separate Emotion Recognition systems:

1. **Speech Emotion Recognition (SER)** — classifies emotion from raw `.wav` audio using a WavLM transformer fine-tuned on the TESS dataset with a speaker-aware training split.
2. **Text Emotion Recognition (TER)** — classifies emotion from text utterances using a RoBERTa transformer fine-tuned on the DailyDialog dataset with a dialogue-level leakage-free split.

Both pipelines predict the same **7 emotion classes**, include GUI and batch inference tools, and produce complete metrics for evaluation.

## Datasets

### TESS — Toronto Emotional Speech Set (Speech Pipeline)

- **Speakers:** OAF (Older Adult Female), YAF (Young Adult Female)
- **Utterances:** ~200 single-word utterances × 7 emotions × 2 speakers ≈ 2,800 files
- **Format:** `.wav`, 16 kHz, studio-recorded
- **Balance:** Perfectly balanced across all 7 classes per speaker
- **Split:** OAF → primary training base; YAF → 14% adapt per class added to training; remaining YAF → 30% val / 70% test
- **Limitation:** Controlled environment; does not represent spontaneous real-world speech

Place TESS folders under `dataset/` as `dataset/OAF_angry/`, `dataset/YAF_happy/`, etc.

### DailyDialog (Text Pipeline)

- **Dialogues:** 13,118 multi-turn English conversations
- **Utterances:** 102,965 total utterances
- **Balance:** Highly imbalanced — neutral ≈ 83% of all utterances
- **Split:** 9,181 train / 1,968 val / 1,968 test dialogues (by `dialogue_id`)
- **Leakage prevention:** `GroupShuffleSplit` on `dialogue_id` ensures no dialogue crosses splits
- **Limitation:** Written conversational style; minority classes (fear n=32, disgust n=50 in test) have insufficient support for reliable per-class metrics

Place `dialogues_text.txt` and `dialogues_emotion.txt` inside `models/text_pipeline/data/`.

## Architecture Decisions

### Speech Pipeline

#### Preprocessing Block
Raw `.wav` files are loaded at 16 kHz, trimmed of silence (`top_db=30`), cropped/padded to exactly 3 seconds (48,000 samples), and peak-normalized. **Trade-off:** Fixed 3-second window discards content in longer utterances but ensures consistent model input shape.

#### Feature Extraction Block
`AutoFeatureExtractor` from `microsoft/wavlm-base` normalizes raw waveforms to zero-mean, unit-variance before feeding into WavLM. This runs per-sample inside the DataLoader. **Trade-off:** Correct for transformer expectations, but no caching — re-computed each epoch, which is slow.

#### Temporal Modelling Block
`microsoft/wavlm-base` — a 12-layer transformer pretrained on 960h of unlabeled speech. Self-attention captures long-range temporal dependencies in the waveform. Mean pooling over all time steps of `last_hidden_state` (shape: `[B, T, 768]`) produces a single 768-dim utterance vector. **Trade-off:** Mean pooling is simple and effective; attention-weighted pooling could be stronger.

#### Classifier Block
`Dropout(0.3) → Linear(768→256) → ReLU → Dropout(0.3) → Linear(256→7)`. Two-stage reduction with dropout regularization. CrossEntropyLoss with label smoothing 0.03 and mild class weights for the balanced TESS dataset.

---

### Text Pipeline

#### Preprocessing Block
DailyDialog lines are split by `__eou__` delimiter into utterances; emotion IDs are mapped to labels; utterances ≤ 2 chars are dropped; duplicates removed by `(dialogue_id, utterance_id, text, emotion)`. `GroupShuffleSplit` on `dialogue_id` enforces leakage-free splits. **Trade-off:** Dialogue-level split may produce slightly uneven class distributions between splits.

#### Feature Extraction Block
`AutoTokenizer` from `roberta-base` tokenizes the full dataset once before training (`tokenize_dataframe()`), with `max_length=96`, padding, and truncation. Tokens stored as tensors — no re-tokenization per batch. **Trade-off:** Memory-efficient and fast; 96 tokens covers ~95% of DailyDialog utterances.

#### Contextual Modelling Block
`roberta-base` — 12-layer transformer pretrained on BooksCorpus + Wikipedia + OpenWebText. Bidirectional self-attention models word context across the full utterance. Attention-mask-weighted mean pooling produces a 768-dim utterance embedding. **Trade-off:** RoBERTa captures rich semantics but is heavy (125M params); distilRoBERTa could be faster with minor accuracy loss.

#### Classifier Block
Identical structure to speech: `Dropout(0.3) → Linear(768→256) → ReLU → Dropout(0.3) → Linear(256→7)`. CrossEntropyLoss with dynamic class weights (capped at 2.5×), label smoothing 0.03, and a linear warmup + decay scheduler to stabilize RoBERTa fine-tuning.

### Fusion
Fusion is **not implemented** in this version. Both pipelines are evaluated independently. Multimodal fusion is planned as future work (see section below).

## Analysis

### A. Easiest and Hardest Emotions to Classify

#### Speech Pipeline

| Difficulty | Emotions | Reason |
|---|---|---|
| Easiest | anger, disgust, fear, neutral, sadness | Perfect F1=1.00 — distinct acoustic features in TESS controlled recordings |
| Near-perfect | happiness, surprise | F1≈0.996 — 1 misclassification each; both are high-energy positive/arousal emotions with overlapping pitch contours |
| Hardest | happiness ↔ surprise | Only 2 cross-class errors in 843 samples; in TESS these share upbeat prosody |

#### Text Pipeline

| Difficulty | Emotions | Reason |
|---|---|---|
| Easiest | neutral (F1=0.887) | Dominant class (83% support); model defaults to neutral for ambiguous text |
| Good performance | happiness (0.574), surprise (0.550) | Distinctive positive or exclamatory lexical patterns |
| Hardest | disgust (F1=0.262), sadness (0.362), anger (0.379) | Very small test support; lexical overlap with neutral; similar negative valence makes them hard to separate |
| Severe confusion | anger, sadness, disgust → predicted as neutral | 89 anger, 185 sadness, 46 disgust predicted as neutral in the confusion matrix |

### B. Fusion Impact

Multimodal fusion is not implemented. Speech and text pipelines are evaluated independently. Fusion is listed under Future Work.

### C. Error Analysis — Realistic Failure Cases (Text Pipeline)

The following cases are drawn from `metrics/batch_text_predictions.csv`:

| Input | Final Label | Top-1 | Top-2 | Top-3 | Analysis |
|---|---|---|---|---|---|
| `Okay, I will come there tomorrow.` | **uncertain** | happiness (43.4%) | neutral (39.2%) | surprise (3.8%) | Gap = 4.2% — well below 8% threshold. Neutral short-statement, no clear emotion signal. Classic DailyDialog ambiguity. |
| `I am scared something bad might happen.` | fear (69.4%) | fear | neutral (14.3%) | sadness (7.4%) | Correctly predicted but notable neutral pull (14%) — short fearful utterances bleed into neutral domain |
| `That behavior is disgusting and unacceptable.` | disgust (62.8%) | disgust | anger (30.5%) | neutral (2.2%) | Correct but anger is very close (30.5%). Disgust and anger share negative high-arousal semantics — common confusion pair in DailyDialog |

**Structural failure pattern (from confusion matrix):**
- 1,571 happiness utterances → predicted neutral (81% of happiness errors)
- 185 sadness utterances → predicted neutral
- 89 anger utterances → predicted neutral

The model's strong neutral prior (trained on 83% neutral data) consistently absorbs ambiguous or mild emotional signals.

**Recommended failure-case extraction:** Run `batch_test.py` on test split samples with known ground truth, then filter for `final_label ≠ true_label` to build `metrics/text_failure_cases.csv`.

- **Text (RoBERTa):** Neutral cluster will dominate the embedding space. Disgust, anger, and sadness will appear close together (shared negative valence). Fear may form a small separate cluster.

## GUI and Testing

### Speech GUI (`speech_pipeline/test.py`)

- **Input:** Upload `.wav` file (any format librosa supports) or hold-to-record via microphone
- **Preprocessing:** Trim → pad/crop to 3s → normalize → WavLM feature extraction
- **Output:** Predicted emotion label, confidence score, progress bar, emoji
- **Uncertainty rule:** If top-1 confidence < 55%, displays `UNCERTAIN`
- **Reports:** Buttons to view classification report, confusion matrix, metrics summary, and training plots

### Text GUI (`text_pipeline/test.py`)

- **Input:** Free-text sentence typed into the GUI textbox
- **Preprocessing:** URL removal, whitespace normalization, RoBERTa tokenization
- **Output:** Top-3 predicted emotions with confidence scores
- **Uncertainty rule:** If top-1 confidence < 50% **OR** gap between top-1 and top-2 < 8%, displays `UNCERTAIN`
- **Reports:** Buttons to view classification report, confusion matrix, training curve, metrics JSON

## Data Leakage Prevention

### Speech Pipeline
- Split is **speaker-aware**: OAF speaker forms the training base; YAF utterances are divided into adapt (14%/class) + val (30%) + test (70%)
- No audio file appears in more than one split
- Prevents speaker identity leakage — model must generalize from OAF to YAF

### Text Pipeline
- Split uses `sklearn.model_selection.GroupShuffleSplit` on `dialogue_id`
- **Guarantee:** No utterance from the same dialogue appears in more than one split
- Leakage check is run programmatically via `check_dialogue_leakage()` — raises `ValueError` if any `dialogue_id` overlaps across splits
- Verified splits: 9,181 train / 1,968 val / 1,968 test dialogues (confirmed in `processed_data/split_info.json`)

## Limitations

1. **Speech accuracy is dataset-specific.** TESS has only 2 speakers in a studio environment. 99.88% accuracy on YAF-only test data reflects near-perfect in-distribution learning, not generalizable SER. Real-world performance will be significantly lower.
2. **DailyDialog class imbalance.** The neutral class makes up ~83% of all utterances. This inflates accuracy (81.3%) and weighted F1 (82.8%). Macro F1 (49.77%) is the honest metric.
3. **Disgust and fear have tiny test support.** Only 50 and 32 test samples respectively — per-class F1 estimates for these classes have high variance and are not statistically reliable.
4. **Text model overfits progressively.** Validation loss increases from epoch 1 to 8 while training loss decreases. Early stopping at patience=2 helped prevent worse overfit.
5. **No feature caching for speech.** WavLM feature extraction runs inside `__getitem__` on every epoch — no `.npy` cache is used, making training slower than necessary.
6. **No multimodal fusion.** Speech and text pipelines operate completely independently.
7. **No representation visualization.** t-SNE/PCA plots of learned embeddings are not generated automatically.

## Future Work

- **Multimodal fusion:** Combine WavLM and RoBERTa embeddings using confidence-weighted, learned, or attention-based fusion for richer emotion prediction
- **Real-world speech dataset:** Test SER on RAVDESS, IEMOCAP, or EmoDB to measure generalization beyond TESS
- **Feature caching:** Pre-compute and save WavLM embeddings to `processed_data/*.npy` to eliminate redundant per-epoch extraction
- **Better class balancing (text):** Explore undersampling neutral, oversampling minority classes, or focal loss to improve disgust/fear recall
- **Representation visualization:** Implement t-SNE/PCA plots of encoder embeddings colored by emotion class for qualitative analysis
- **Cross-corpus text evaluation:** Evaluate RoBERTa model on MELD or IEMOCAP text transcripts to test generalization beyond DailyDialog
- **Confidence calibration:** Investigate whether model confidence scores are well-calibrated using reliability diagrams

## Conclusion

This project successfully delivers two fully-trained emotion recognition pipelines:

- The **Speech pipeline** uses `microsoft/wavlm-base` with speaker-aware training on TESS, achieving 99.88% Macro F1 on the in-distribution YAF test set. This result demonstrates WavLM's strong feature representation for emotion-labelled speech, though it is specific to the controlled TESS dataset.

- The **Text pipeline** uses `roberta-base` with dialogue-level leakage-free training on DailyDialog, achieving 49.77% Macro F1 on a 15,581-utterance test set. This is consistent with published results on DailyDialog for 7-class emotion classification, where class imbalance (83% neutral) is the primary challenge.

Both pipelines include proper data leakage prevention, gradient clipping, early stopping, checkpointing, GPU support, complete evaluation metrics, and working GUI/batch inference tools suitable for demonstration.
